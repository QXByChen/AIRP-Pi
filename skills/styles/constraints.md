# 文本生成约束（硬性）

本文件为 AIRP-Pi 的反八股强约束规则集。Pi Agent 每轮生成前必须内化这些规则。
违反任何一条即视为不合格输出。

---

## 一、绝对禁用词库

### 1.1 禁用句式（零容忍）

| 句式 | 说明 |
|------|------|
| ——（破折号） | 正文中绝对禁止使用，无论任何语境 |
| 不是……而是…… | 先否后肯句式全部禁止 |
| XX……不，是…… | 自我纠正式转折禁止 |
| 但是/然而 + 开头 | 禁止用"但是""然而"作为句子或段落开头 |
| 就像/如同/仿佛 + 动物 | 将人比作动物的比喻全部禁止 |
| 像在陈述一个事实 | 禁止 |
| 强调语气就像XXX时一样 | 禁止 |
| 先否后肯的所有变体 | 包括"与其说是X不如说是Y" |

## 二、抗缺陷指南

### 2.1 角色扁平化
**问题**：把"冷静/沉默/高冷"角色写成无情绪机器人，全程面无表情。
**解决**：冷静的人也有情绪，只是表达方式不同。沉默中的手指动作变化、视线偏移方向、呼吸节奏微调、选择性回避话题——这些才是内敛角色的情绪出口。

### 2.2 "自然"滥用
**问题**：所有动作都"自然地""随意地"完成，实际上是在逃避描写。
**解决**：删掉"自然"二字，裸写动作本身。"她自然地坐下" → "她拉开椅子坐下，膝盖碰到桌腿"。

### 2.3 情感平板
**问题**：无论什么场景都用同一种温度推进，缺乏情绪色彩。
**解决**：每个场景必须有明确的情绪基调。通过环境细节（光线变化、声音远近、温度感知）和角色微动作传达，而非直接标签。

### 2.4 角色跪舔
**问题**：NPC 失去独立人格，无条件配合玩家、讨好玩家、崇拜玩家。
**解决**：角色有自己的立场、底线、不耐烦的时刻。会拒绝、会走神、会因为自己的事情分心。关系亲密不等于无条件服从。

### 2.5 对话失真
**问题**：台词极端化（过于精确/恶意/油腻/文艺），不像真人说话。
**解决**：对照角色的年龄、教育背景、当前情绪状态、与对方的关系远近。真人说话会犹豫、会用错词、会说到一半改口。

### 2.6 逻辑断裂
**问题**：时间/对话/行为/物品状态自相矛盾。
**解决**：每轮开始前检查——上轮结束时各角色在哪、穿什么、手里拿什么、说到哪了。物理状态必须连续。

### 2.7 段落格式重复
**问题**：每段都是"他/她XXXX"开头，形成机械节奏。
**解决**：变换开头方式——环境切入、动作切入、对话切入、感官切入、时间切入。同一段内主语位置灵活调整。

### 2.8 性场景过度害羞
**问题**：NSFW 场景中角色突然变成处女反应，无论其设定经验如何。
**解决**：按角色实际经验和性格写。老手不会每次都脸红；主动的人不会突然变被动；熟悉的伴侣之间有默契而非每次都像初次。

### 2.9 职业刻板印象
**问题**：医生=冷静理性、军人=粗暴直接、教师=温柔耐心、商人=精明算计。
**解决**：人格独立于职业。暴躁的医生、温柔的军人、刻薄的教师都是合理的。职业只影响知识储备和习惯动作，不决定性格。

## 三、抗重复机制

- 禁止与上一轮使用相同的：段落开头方式、结尾方式、核心比喻、句式结构
- 禁止连续两轮出现相同的环境描写角度（如连续两轮都写光线）
- 禁止连续两段使用同一感官通道（连续两段都写气味/都写声音/都写触觉）
- 禁止重复使用同一个动词超过 3 次（同一轮内）
- 对话中禁止两个不同角色使用相同的口头禅或语气词

## 四、抗逻辑错误

常见逻辑错误类型（生成前必须检查）：

- **物理矛盾**：已脱掉的衣服不能再次脱掉；已放下的东西不能再放下
- **空间矛盾**：角色不能同时在两个地方；移动需要时间
- **时间矛盾**：5 分钟内不能做 2 小时的事；对话长度要匹配时间流逝
- **对话矛盾**：刚说过的话不能当没说过；已回答的问题不能再问
- **状态矛盾**：受伤的部位不能突然好了；醉酒状态不能突然清醒
- **感知矛盾**：背对着的人看不到对方表情；隔着墙听不到耳语

---

## 五、抗全知视角

角色必须通过第一人称有限视角感知世界：
- 不能直接知道世界设定或其他角色的内心想法
- 必须通过看、听、推理、回忆等感知过程获取信息
- 角色对他人动机的判断可以是错的
- 未亲眼见到的事件只能通过转述/推测获知

---

## 六、抗平淡

- 每个场景必须有"浓墨重彩"的一笔——一个让人记住的细节、动作或对白
- 角色的行动必须带有情感色彩，不能像在填表
- 日常场景也要有质感——通过感官细节让平凡时刻变得可触摸
- 禁止"流水账"式推进：起床→洗漱→出门→到达。必须选择性聚焦
